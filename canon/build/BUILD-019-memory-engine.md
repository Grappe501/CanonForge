## 019.0 — Purpose

The Memory Engine defines how **experience becomes memory**, how **memory becomes meaning**, and how **meaning reshapes identity**.

Chronoforge is not interested in perfect recall. It is interested in **living memory**.

Every reread should feel like remembering your own life at a different age. **The events are the same. You are not.**

*(CF-CON-416 · CF-CON-417 Memory as reconstruction)*

Every important event exists simultaneously in **five forms**. *(CF-CON-419)*

| State | Name | Governs |
|---|---|---|

| **5** | **Integrated** | Memory becomes identity — no longer merely remembered · part of how one sees the world |

## One text · changing reader memory *(CF-ENTRY-057 · CF-CON-430)*

The novel does not have five readings. It has **one text**.

What changes is the **reader's memory of the text**:

| Read | What shifts |
|---|---|
| **2** | Opening haunted by memories of the ending |
| **3** | Scenes carry echoes from relationships now understood |
| **5** | Conversation between printed words and accumulated reader memory |

The true medium of Chronoforge is not paper · ink · or language.

It is the **evolving relationship between fixed text and the reader's changing memory**.

Compose for that relationship **deliberately** — reread is design from the **first sentence**, not afterthought.
