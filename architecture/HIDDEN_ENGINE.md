# Hidden Engine

> Five simultaneous systems operating beneath the adventure surface

---

## Overview

The novel runs **five hidden systems** concurrently on every page. Read 1 experiences only System 0 (surface adventure). Each Lock reveals the next system.

```txt
SYSTEM 0  →  Adventure surface (visible)
SYSTEM 1  →  Chronology Lie + time fractures
SYSTEM 2  →  Character dual chronology
SYSTEM 3  →  World/mechanics hidden layer
SYSTEM 4  →  Meta-narrative / reader contract
SYSTEM 5  →  Reader-as-protagonist (sixth protagonist)
```

---

## System 1: Chronology

- Chapters ordered by **causality/thematic chain**, not time
- Adjacent movements may span decades or centuries
- Five **time fractures** map to genre pillars (1905→1977)
- Time anchors embedded per movement

---

## System 2: Character

- Every POV character carries **beliefs vs truth** (dual chronology)
- Crash-style convergence: separate threads → one mechanism
- Five protagonists fork A/B (open — CF-CON-005)

---

## System 3: World

- Surface: planetary romance / space opera grammar
- Depth: world rules reframe on Lock 3
- Originality firewall: techniques from pillars, not copied worlds

---

## System 4: Story type

- Read 1 genre assumption ≠ final genre understanding
- Butterfly Effect: small moments carry temporal weight
- LOST breadcrumbs: finite, fair, solvable

---

## System 5: Reader

- Reader is sixth protagonist
- Opening as continuation of reader's life
- First/last sentence dyad
- Invitation ending; meaning in interpreter

---

## Candidate apex

**Memory not time** (CF-CON-006) — may unify Systems 1 and 5. Status: candidate, not locked.
