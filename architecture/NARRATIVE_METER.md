# Narrative Meter

> CF-CON-101 · CF-ENTRY-030 · complements [THE_SCORE.md](THE_SCORE.md)

---

## Mechanical vs organic (CF-CON-102)

| Mechanical (reject) | Organic (pursue) |
|---|---|
| 11 words per line | Pulse serves story |
| 247 words per page | Rhythm emerges from demand |
| Every page ends with noun | Measure serves music |
| Fixed paragraph sentence count | Tension · release · syncopation · silence |

**Do not** specify exact words per line/page — that becomes a **prison** and breaks across editions.

Percussionist discipline: learn pulse and release, not "every measure has exactly X notes."

## Page fingerprint (CF-CON-110)

Every page receives weighted attributes (1–10 scale · Phase 4 scoring):

| Attribute | Example | Meaning |
|---|---|---|
| **Pulse** | 4 | Underlying beat strength |
| **Density** | 7 | Visual/textual mass |
| **Silence** | 2 | Rest weight on page |
| **Emotional pressure** | 8 | Cumulative tension |
| **Discovery** | 3 | New information surface |
| **Reader certainty** | 9 | Apparent confidence |
| **Reader certainty (true)** | 2 | Actual confidence |

The gap between apparent and true certainty = **Recognition fuel**.

Fingerprint ≠ word count. Fingerprint = **weight map**.

## Breathing cycles (CF-CON-104)

Design **emotional phrases**, not page counts.

```txt
Every ~12–15 pages → reader subconsciously completes one emotional phrase
Resolution by accumulated rhythm — not page number
Survives: trim size · font · ebook reflow · translation
```

## Publication-survivable anchors (CF-CON-109)

**Anchor meaning to:**

- Sentences · paragraphs · scenes
- Recurring motifs · punctuation · typography
- Emotional cadence · breathing cycles

**Never anchor to:**

- Line 14 · page 37 · word 212

Fragile across publisher trim · font · ebook · translation.

SURFACE ARCHITECTURE          DEEP ARCHITECTURE

## Reread tempo (CF-CON-107)

Same text · different **performance** each read:

| Read | Tempo | Analog |
|---|---|---|
| 1 | Adagio | First rehearsal |
| 2 | Andante | Learning patterns |
| 3 | Moderato | Anticipating cadence |
| 4 | Allegro | Recognition accelerating |
| 5 | Presto | Concert |

Reader recognizes patterns sooner — text invariant.

---

## Reader performance model (CF-CON-106)

```txt
BOOK    = score
READER  = performer
READ 1  = first rehearsal
READ 5  = concert
```

Job: compose experience that **matures as performed** — not write a "perfect story" frozen at print.

Pairs with Master Canon:

> **The novel is not complete when it is printed.**
> **The novel is completed each time it is read.** (CF-CON-108)
